1. System Scope

This dashboard is a deterministic technical monitoring system implemented in pure HTML5 and JavaScript. All analytics are computed client-side. No backend services are required.

All quantities are deterministic transformations of the observed closing price series \(P_t\). No alternative parameterizations or smoothing conventions are implemented unless explicitly stated.

2. Data Architecture

Data is sourced from Stooq CSV endpoints:

  • Daily: https://stooq.com/q/d/l/?s=<symbol>&i=d
  • Weekly: https://stooq.com/q/d/l/?s=<symbol>&i=w

Symbols must be entered in Stooq-compatible format (e.g., voo.us).

3. Indicator Definitions

Closing Price

Plain Language: The observed closing price at time t.

Statistical Description: A univariate time-indexed stochastic process observed discretely.

\[ P_t \]

Financial Terminology: Closing price.

Long-Horizon Trend (40-period SMA)

Plain Language: Average price over the most recent 40 observations.

Statistical Description: A fixed-window rolling mean estimator acting as a linear smoothing operator.

\[ T_{40,t} = \frac{1}{40} \sum_{i=0}^{39} P_{t-i} \]

Financial Terminology: 40-period Simple Moving Average.

Trend Slope

Statistical Description: First discrete difference of the rolling mean estimator.

\[ \Delta T_{40,t} = T_{40,t} - T_{40,t-1} \]

Exponential Moving Averages

Statistical Description: Recursive exponential smoothing estimators with fixed decay parameters.

\[ T_{26,t} = \alpha_{26} P_t + (1-\alpha_{26})T_{26,t-1}, \quad \alpha_{26}=2/27 \]

\[ T_{12,t} = \alpha_{12} P_t + (1-\alpha_{12})T_{12,t-1}, \quad \alpha_{12}=2/13 \]

Deviation

Statistical Description: Linear difference between two exponential smoothing estimators.

\[ D_t = T_{12,t} - T_{26,t} \]

RSI (14, Wilder)

Statistical Description: Bounded nonlinear transformation of the ratio of exponentially smoothed positive and negative price-change magnitudes.

\[ RSI_t = 100 - \frac{100}{1 + RS_t}, \quad RS_t = \frac{AvgGain_t}{AvgLoss_t} \]

4. Deterministic State Machine

Eligibility Filter

\[ E_t = 1(P_t > T_{40,t} \land \Delta T_{40,t} > 0) \]

Tactical States

  • T1: \(D_t \ge 0 \land RSI_t \ge 50\)
  • T2: \(D_t \ge 0 \land RSI_t < 50\)
  • T3: \(D_t < 0 \land RSI_t \ge 50\)
  • T4: \(D_t < 0 \land RSI_t < 50\)

5. Concordance & Run-Length

\[ C_t = 1 \text{ if } (D_t \ge 0 \land RSI_t \ge 50) \lor (D_t < 0 \land RSI_t < 50) \]

Run-length counter:

\[ k_C(t) = \begin{cases} k_C(t-1)+1 & \text{if } C_t=1 \\ 0 & \text{otherwise} \end{cases} \]

6. Alert Classification (Tab 1)

  • Tier A: \(E_t=1\), T1, \(k_C \ge 3\)
  • Tier B: Fresh transition into T1
  • Tier C: Eligible but discordant

7. Data Integrity & Indeterminate State

Indeterminate state is triggered when:

  • Insufficient lookback window
  • Malformed or missing data
  • Failed data fetch

8. User Interface Architecture

  • Left Sidebar: Overview (Tab 1), Monitoring (Tab 2)

  • Concurrency-limited Refresh All
  • Per-row Fetch

9. Implementation Constraints

  • Client-side only
  • LocalStorage caching
  • No automatic data fetch on first load
  • Watchlist cap: 15 symbols
